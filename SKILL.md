# 公务员行测图形推理解题 Skill v2

## Core Principle

图形推理不能看到一个“像的规律”就直接下结论。必须按：

```text
识别信号 -> 优先考点 -> 验证方式 -> 常见误判提醒
```

来解题。核心目标不是记更多考点，而是让 AI 像图推老师一样：先观察特征，再筛选考点，再验证规律，再排除选项，最后总结识别信号。

必须遵守：

1. 先看元素组成是否相同。
2. 元素相同，优先考虑位置、样式、叠加、遍历、黑白运算。
3. 元素不同，先看是否有明显属性规律。
4. 无明显属性规律，再转向数量规律。
5. 出现明显封闭区域，优先考虑面。
6. 出现明显线条、出头、交叉、曲直混合，优先考虑线、点、一笔画。
7. 出现黑白块，不能只看黑白运算，还要看黑/白数量、连接块、相接边、公共边、直角、面积、种类数、对称轴。
8. 外框一致且内部复杂，外框通常不是主规律，优先看内部面、点、线、交点、对称轴关系。
9. 看似旋转、移动、对称的题，必须检查是否存在更可量化的面、线、点、边、直角、黑白运算规律。
10. 无法验证全部题干图形时，必须标注不确定，不能硬猜答案。

## Question-Type First

先判断题型，再套规则：

- 一组式：通常 5 图 + 问号，找连续规律。
- 两组式：前 3 图一组，后 2 图 + 问号一组，用第一组规律套第二组。
- 九宫格：优先横看、竖看、整体看、中心看、对角线看、米字型看。
- 分组分类：找 6 图的共同属性，分成两组，不要先找递增。
- 空间重构：优先看相对面、公共边、公共点、折叠方向、可见面、图案方向。
- 文字数字图推：汉字、字母、数字、符号优先看笔画、面、线、点、开闭、曲直、对称。
- 不确定：图像细节不足、选项缺失、规则不唯一时明确标注。

## First-Eye Feature Scan

第一眼必须描述最明显特征：

- 元素组成相同 / 不同。
- 外框是否一致。
- 封闭区域是否明显。
- 黑白块 / 黑白球 / 阴影是否明显。
- 线条是否复杂，是否曲直混合。
- 是否规整、对称、像旋转或移动。
- 是否像立体图、展开图、截面图、三视图。
- 是否有出头线、悬空点、交叉线、标记点。
- 是否有圆、三角形、条纹、重合区域、相切/相交关系。

至少列出 2-3 个候选考点，再说明为什么筛掉其中一些。

## Trigger Signals and Priority

### 元素组成相同

优先考虑：

1. 位置变化。
2. 平移。
3. 旋转。
4. 翻转。
5. 样式叠加。
6. 黑白运算。
7. 元素遍历。
8. 内外圈移动。
9. 行列移动。
10. 对角线移动。

提醒：

- 翻转不只有左右翻，也有上下翻。
- 看似旋转，可能实际是黑白运算。
- 移动路径找不到时，改数每行、每列、每区域、每象限的数量。
- 多元素题先给位置编号，再建立固定映射表。

### 元素组成不同

优先考虑：

1. 属性规律。
2. 数量规律。
3. 面。
4. 线。
5. 点。
6. 角。
7. 对称。
8. 一笔画。
9. 开闭性。
10. 曲直性。

提醒：

- 不要凭整体相似判断。
- 有封闭区域，优先数面。
- 线条复杂，优先看线和点。
- 图形凌乱，优先看数量规律。

## Symmetry Rules

### 识别信号

看到这些情况，优先考虑对称：

- 图形整体规整。
- 多个图形都很规则。
- 内外结构明显。
- 黑白小球呈镜像分布。
- 条纹或内部结构规整。
- 选项看不出明显数量规律。

### 检查顺序

对称不能只判断“是否对称”，必须继续查：

1. 轴对称、中心对称、轴 + 中心对称、非对称。
2. 对称轴方向：横、竖、斜、旋转变化。
3. 对称轴数量：相同、递增、递减、0 条。
4. 对称轴是否过点、过线、过面。
5. 对称轴是否与内部线条重合、垂直、平行、相交。
6. 对称轴穿过几个面、几个黑球/白球。
7. 内外图形是否要分开看。
8. 黑白块、黑白球是否要分开看对称轴。

重要补充：

- 非对称不是“没有规律”，可视为对称轴数量为 0。
- 整体很对称且都是封闭图形时，可检查“对称轴数量与面数量”的关系。
- 内外图形可分别数对称轴，再做差或比。

误判提醒：

- 对称不稳时，不要硬判对称，转查直线数量、内部线条数量、轴线与内部线关系。
- 伪装对称题可能实际考线数量。

## One-Stroke and Stroke Count

### 识别信号

看到这些特征，考虑一笔画 / 多笔画：

- 点连接图形。
- 出头线明显。
- 线条端点明显。
- 圆形相交、圆相切。
- 日字、田字、五角星、多圆相交变形。
- 多条交叉线、混乱曲线。
- 存在间断线条。
- 多部分分离图形。
- 汉字、字母、符号。

### 判断规则

- 一笔画题要看奇点数，不能只看整体。
- 多部分图形要分开算笔画数。
- 有几个独立部分，通常至少几笔。
- 不规则内外图形，可去掉外轮廓看内部几笔。
- 一笔画可结合曲直性、连接方式、框内线数考查。

误判提醒：

- 开闭、一笔画、直角常可作为最后兜底考点。
- 不要一看到出头线就下结论，必须验证全部题干。

## Face Rules

### 识别信号

看到这些特征，优先考虑面：

- 图形被分割。
- 封闭区域明显。
- 外框一致、内部线条复杂。
- 生活化粗线条图形且有明显留白区域。
- 圆内有线条。
- 三角形较多。
- 纯封闭图形。
- 线构空间。
- 元素不同且无明显属性规律。
- 复杂程度不单调。

### 判断原则

1. 外部凌乱看内部。
2. 外部规则也必须看内部。
3. 内外都乱整体看。
4. 内外分离结合看。
5. 外部相同通常看内部。
6. 小面明显但规律不对时，看最大面、最小面、最大面积。
7. 纯封闭图形不只看总面数，还可看内部线条数。

### 常见考法

- 总面数。
- 三角形面、四边形面、特殊面。
- 最大面 / 最小面形状或面积。
- 相同面数量、面种类数、面的边数。
- 与外框形状相同的面。
- 相交面、相交面的边数。
- 面的公共边数量、对接方式。
- 面数量与外框边数、交点数、对称轴数、线条数复合。
- 面数量作差、加和、行列定值。
- 九宫格面数横向/纵向加和。

三角形多时，不要只数三角形个数，还要看：

- 三角形面数量。
- 三角形面的形状。
- 三角形和四边形作差。
- 特殊三角形面。
- 三角形与外框关系。

## Line Rules

### 识别信号

看到这些特征，考虑线：

- 线条很多。
- 外框一致但内部线不同。
- 纯直线图形。
- 曲直混合。
- 平行线明显。
- 火柴棒、电子数字、回旋折线。
- 延伸线、切线、割线。
- 对角线、条纹明显。

### 常见考法

- 直线数、曲线数。
- 曲线 + 直线求和，或作差。
- 横线、竖线、平行线组数、平行线方向。
- 对角线数量、方向、对角线上交点数。
- 延伸线数量，以及从点延伸还是从线延伸。
- 切线、割线数量和关系。
- 回旋折线首尾方向，首尾平行/垂直关系。
- 外框线条数与内部线条数加和/作差。

提醒：

- 火柴棒题首选数量。
- 五边形可能经常考直线数。
- 如果线很多但找不到规律，考虑数点。
- 看似对称但不稳时，可以数直线。

## Point and Angle Rules

### 识别信号

看到这些特征，考虑点或角：

- 交叉线明显。
- 曲直混合。
- 圆内有线。
- 切线、割线明显。
- 出头线、悬空点、标记点明显。
- 外轮廓相同，内部线变化。
- 黑点、白点、小圆点明显。
- 立体图形有明显顶点。
- 有矩形、垂直关系、小直角三角形。

### 点的考法

- 总交点、内部交点、外部交点。
- 圆内交点、曲直交点、切点。
- 线上交点、拐点。
- 标记点数量、标记点引出的线条数。
- 标记点在线端、线中、交点中。
- 轮廓内点和轮廓外点做运算。
- 立体图形顶点数。

### 角的考法

- 角数量。
- 直角数量。
- 锐角数量。
- 内角 / 外角。
- 黑块内部直角。
- 火柴棒图形看直角。

提醒：

- 曲线和直线相交明显时，优先考虑曲直交点。
- 常规点数不出规律时，考虑特殊点。
- 角默认不重复数；若选项全不匹配，再检查重复数角。

## Element Rules

### 识别信号

看到这些特征，考虑元素：

- 多个小元素分离。
- 元素种类明显。
- 相邻图形有重复元素。
- 生活化简笔画。
- 元素开闭差异。
- 元素之间相离、相接、相切、相交、重合。
- 元素嵌套。
- 元素在不同位置移动。
- 黑白元素、符号元素、功能元素明显。

### 常见考法

- 元素种类、元素数量、相同元素数量。
- 元素传递、元素遍历。
- 曲直元素、开闭元素、部分数。
- 元素对接方式、边数、嵌套层数。
- 元素移动、转换、换算。
- 功能元素方向，符号元素位置。
- 黑白元素数量、位置、对称。

提醒：

- 所有元素分离且相邻图有相同元素时，看元素传递。
- 生活化简笔画可考虑部分数。
- 元素移动可按行列、内外圈、斜线、象限、对角线。
- 多个元素可能抱团移动，也可能各自移动。

## Position Rules

元素相同或高度相似，优先考虑位置变化。

常见考法：

- 上下左右平移。
- 绕圈移动。
- 顺时针/逆时针旋转。
- 旋转角度。
- 左右翻转、上下翻转。
- 内外圈移动。
- 行列移动、斜线移动、象限移动、对角线移动。
- 多元素同时移动、抱团移动、整体移动。
- 功能元素方向变化。

误判提醒：

- 不要只说“旋转”，还要排查翻转。
- 看似移动但路径找不到时，数行列/区域数量。
- 看似旋转可能是黑白运算。
- 中间元素的移动不能忽略。

## Style and Black-White Operations

### 识别信号

看到这些特征，考虑样式规律：

- 图形结构相似但黑白变化。
- 九宫格中图案叠加感明显。
- 黑白块位置变化。
- 阴影区域变化。
- 两图合成第三图。
- 黑白区域有去同存异特征。

### 常见考法

- 加法、减法。
- 求同、求异。
- 去同存异、去异存同。
- 黑白叠加、黑白运算。
- 结合旋转/翻转后的加减同异。
- 横向运算、纵向运算。
- 空白区域运算。

提醒：

- 外框不一样也可能是空白区域运算。
- 黑白运算可以横看，也可以竖看。
- 不要被方向变化骗了，先检查黑白关系是否可运算。

## Black-White Specialized Rules

看到黑白块明显、黑色面积大、黑白分割明显、黑白小球明显时，进入黑白专项。

常见考法：

- 黑块数量、白块数量、差值、倍数。
- 黑色连通块数量、白色连通块数量。
- 黑白块相接边条数。
- 黑块与白块相连边数量。
- 黑块与几个白块相邻。
- 黑块与外轮廓公共边数量。
- 黑块外轮廓边数、周长、面积、大小、种类数。
- 黑块直角数量。
- 黑白块对称轴。
- 黑白小球与对称轴关系。
- 黑白运算、黑白块位置变化。

提醒：

- 黑乎乎一团时，不要只看黑白运算，也要看边、直角、面积、种类数。
- 黑色与白色的内部相接边数量很重要。
- 黑块与多少个白块直接连接也是常见考法。

## Overlap and Intersecting Figures

看到两个或多个图形重叠、相交、覆盖、交错，考虑：

- 重合处形状、面积、边数。
- 非重合处形状、边数。
- 相交面数量、形状、边数。
- 相交面与外轮廓是否一致。
- 两个图形的公共边数量。
- 连接方式。
- 黑面与白面连接关系。

提醒：

- 重合不仅看重合处，还要看非重合处。
- 重合题可能考非重合边。
- 很多线时，也要考虑连接方式。

## Space Rules

### 六面体 / 展开图

遇到“能否折成正方体”“哪项不是同一个纸盒展开图”“多个纸盒组合”时，进入空间重构。

固定流程：

1. 标面。
2. 找共同面或共同图案。
3. 找稳定锚点：黑色方块直角顶点、圆点、特殊三角形顶点、特殊边、图案方向。
4. 统一观察方向，如沿该面顺时针画边。
5. 比较同一条边相邻的图案是否一致。
6. 检查公共点引出的线条数是否一致。
7. 检查公共边是否挨着同一内部图案。
8. 检查相对面关系。
9. 若三项一致、一项不同，选不同项。

必须说明：

- 选择哪个共同图案作为锚点。
- 从哪个点/边开始。
- 按什么方向比较。
- 哪条边或哪个公共点不一致。

提醒：

- 不要只凭整体展开形状相似判断。
- 不要只说“相对面不同”，必须说明锚点和方向。
- 找不到稳定锚点时，标注不确定。

### 三视图

- 判断主视图、左视图、俯视图。
- 小立方体叠加题，结合两个视图还原缺失位置。
- 有颜色时必须核对颜色数量和位置。
- 纯白/纯黑视图是强约束。

### 截面图

- 判断哪个能截出 / 不能截出。
- 有三点时，使用“三个不共线点确定一个平面”。
- 看夹角、边比例、平面经过的面。
- 选非题要说明其他项为什么能截出。

### 立体拼合

- 看能否通过旋转拼成。
- 数相同小模块数量。
- 选非题要验证可拼选项。

## Text, Letter, Number Rules

看到汉字、字母、数字、符号时，考虑：

- 横线数量、竖线数量。
- 点、线、面数量。
- 笔画数。
- 字母数量。
- 开闭性、曲直性、对称性。
- 笔顺、汉字结构。
- 数字中间封闭面数量。
- 电子数字火柴棒数量。

提醒：

- 汉字有几笔，可能就是几笔画。
- 数字中间面数量可能成规律。
- 火柴棒数字首选数量。
- 文字数字也可考对称、开闭、曲直。

## Nine-Grid Rules

九宫格不要只横看。按顺序尝试：

1. 横向：1 + 2 = 3、1 - 2 = 3、叠加、遍历、求同/求异。
2. 纵向：每列是否同理。
3. 中心：中心图是否控制周围图。
4. 对角线：对角线是否成规律。
5. 米字型：对称性或属性是否按米字分布。
6. 整体：9 个图是否共同满足同一属性。

相似图形或重复线条时，先试直接加减同异；不成立再试“旋转/翻转后再运算”。

## Grouping Rules

分组题重点不是找递增，而是找二元共同属性。

优先检查：

- 轴对称 / 中心对称 / 两者兼具 / 非对称。
- 一笔画 / 两笔画 / 多笔画。
- 曲直性。
- 开闭性。
- 元素种类数。
- 内部/外部黑点奇偶。
- 黑白部分数。
- 交点数量或交点位置。
- 直角有无或数量。
- 相离、相接、相切、相交、公共边。
- 总数相同但分布不同。

若“一组有规律，一组无规律”是唯一解，必须标为弱规律。

## Misleading Patterns

### 看似旋转

可能实际考：

- 黑白运算。
- 去同存异。
- 黑白块位置。
- 翻转。
- 面数量。

处理：先检查黑白关系和数量关系，再判断旋转。

### 看似移动

可能实际考：

- 每行数量。
- 每列数量。
- 区域数量。
- 象限数量。
- 元素传递。

处理：移动路径找不到时，不要硬猜方向，改数行列/区域数量。

### 看似对称

可能实际考：

- 直线数量。
- 内部线条数。
- 轴线与内部线条关系。
- 对称轴数量为 0。

处理：对称判断不稳时，数直线和内部线。

### 黑乎乎一团

可能考：

- 黑块外轮廓边数。
- 黑块直角数量。
- 黑块种类数。
- 黑块与白块相接边。
- 黑块面积。

处理：不要只看黑白运算。

### 外框一致

可能考：

- 内部面数量。
- 内部点数量。
- 曲直交点。
- 内部线条数。
- 对称轴与内部线条关系。

处理：外框一致时，不要把外框当主规律，优先看内部。

### 纯封闭图形

可能考：

- 总面数。
- 特殊面。
- 最大面 / 最小面。
- 内部线条数。
- 面与外框关系。

处理：不要只数总面数。

### 圆形图

圆分离时可能考面或曲线；圆相交时可能考面或点。

处理：先区分“分离”还是“相交”。

## Anti-Guessing Rules

每次解题必须遵守：

1. 禁止只凭“看起来像”选答案。
2. 禁止只根据整体相似度选答案。
3. 禁止找到一个弱规律就停止。
4. 必须优先寻找能解释全部题干的强规律。
5. 两个规律都能解释时，优先选择更可量化、更能排除选项的规律。
6. 只能筛掉部分选项、不能解释全部题干的规律，只能作为辅助规律。
7. 必须逐图验证题干。
8. 必须逐项排除选项。
9. 图片不清、细节数不清时，必须标注不确定。
10. 答案正确但理由错误，也视为错误解析，应在反馈中修正。
11. 空间重构题必须说明锚点和方向。
12. 数量题必须写出每幅图对应数量。
13. 黑白题必须说明看的是数量、相接边、面积、运算还是对称。
14. 对称题必须说明对称类型、轴数量、轴方向或轴经过要素。
15. 点线题必须说明数的是普通交点、曲直交点、圆内交点还是标记点。

## Uncertainty Handling

使用【不确定点】主动说明：

- 哪些特征能确定。
- 哪些细节看不清。
- 哪些候选规律仍未排除。
- 最终答案是高置信还是暂定。
- 需要什么图片细节才能确认。

典型不确定场景：

- 选项细节太糊，无法数面、点、线、角。
- 六面体图案方向或公共边看不清。
- 多个规律得到不同答案。
- 题目本身存在争议。
- 页面或用户输入缺少选项信息。

## If Image Input Is Not Actually Available

如果当前运行环境或模型不能直接读取上传图片，不要假装看到了图片。

必须：

1. 说明当前调用无法直接识别图片。
2. 请求清晰截图、选项局部放大图，或用户手动描述关键视觉特征。
3. 若只有 OCR 或文字描述，只能基于已确认细节作答。
4. 把所有缺失视觉信息写入【不确定点】。

## Final Reminder

这个 Skill 的最终目标是：

- 根据图形特征触发考点。
- 按优先级验证规律。
- 不被伪装规律带偏。
- 用可量化规律替代主观相似判断。
- 答案正确但理由错误也要记录并修正。
- 每次错题都沉淀成新的识别信号和防误判规则。
